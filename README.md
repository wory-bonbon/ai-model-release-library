# AI生成モデル リリース・ライブラリ（テンプレート）

画像・動画・3D・音声の生成AIモデルを、**公式一次情報ベース**で管理する単一HTMLのリファレンスツール。
ブラウザで開くだけで動作し、依存ゼロ・サーバー不要。新モデルの追記も見た目の変更も、**AIに渡すだけ**で完結します。

> A single-file, dependency-free reference library for generative AI models (image / video / 3D / audio). Open in a browser. Update via AI.

## デモ

`https://wory-bonbon.github.io/ai-model-release-library/`
ブラウザで開けばそのまま動作します。

## 特長

- **単一HTML/ 依存ゼロ** — ダウンロードしてブラウザで開くだけ
- **検索・カテゴリ絞り込み・ソート** — モデルが増えても探しやすい
- **★お気に入り** — `localStorage` に保存。ファイルを差し替えても `id` 紐付けで星は維持
- **確認ステータス** — official / partial / unverified を色相で識別（公式一次情報主義）
- **AI追記プロトコル内蔵** — ファイル冒頭のコメントに更新ルールが書いてあり、AIがそれに従って追記できる
- **印刷CSS対応**

## 使い方

### 1. そのまま使う
HTML をダウンロードしてブラウザで開く。収録済みのサンプルモデルがそのまま一覧表示されます。

### 2. 新しいモデルを追記する（AI利用）
このHTMLと、追加したいモデルのリリース情報（公式ページのURLやスクショ）をAIに渡し、こう伝えるだけ：

```
このHTMLの追記プロトコルに従って、このモデルを1件追加して完全版を返して。
official_url は公式一次情報を確認してから記入して。
```

AIが `MODELS` 配列の末尾に1件追加して返します（DATA以外は変更しません）。

### 3. 自分好みに作り替える（AI利用）
配色やレイアウトを変えたいときも同様に：

```
このHTMLの <style> だけを変えて、◯◯系の配色にして。
DATAブロック・お気に入りロジック・レンダリング関数・追記プロトコルのコメントは変更しないで。
```

## カスタマイズの注意

- 編集してよいのは `<style>` と、`=== DATA START ===` 〜 `=== DATA END ===` のデータ配列のみ
- レンダリング関数・お気に入り保存ロジック（`FAV_KEY`）は触らない
- 外部フォント等を足すとオフラインで挙動が変わるため、ローカル参照用途ならシステムフォント前提を推奨

## 運用プロンプト集（AIに渡す用）

> *《》で囲んだ箇所* は各自の情報に差し替えてください（下記はデモ用の記入例）。

### 推奨モデルと運用のコツ

| 用途 | 推奨ティア | 理由 |
|---|---|---|
| 新モデル追記＋公式URL確認（メイン） | 高精度（Opus系） | 多数の公式サイト横断検索＋JS整合維持 |
| 軽微な1件追記・文言修正 | 標準（Sonnet系） | 構造が単純なら十分・低コスト |
| 下調べ・告知文の整形のみ | 軽量（Haiku系） | 速い。ただし最終のHTML編集は上位ティアへ渡す |

- **更新は1スレッドに溜めず、こまめに新チャットで完結**。ファイルが育つほど毎ターンの読込コストが上がるため。
- 告知は**テキストで渡す**（Discordの「テキストをコピー」を使う。スクショより軽く、**リリース日も一緒にコピーされる**ので release_date の取りこぼしを防げる）。
- 毎回 **このHTMLを添付**して渡す（構造ルールをファイルから読ませる）。

### A. 初回：活用例（my_usage）を入れて私用版を作る

新チャットに *このHTML* を添付して、以下を貼る：

```
添付HTMLを私個人用の【私用版】にします。各モデルの my_usage に私の活用例を記入してください。

【私のプロフィール】
- 職種：《例：グラフィック／映像クリエイター》
- 主領域：《例：SNS動画・ロゴ・販促バナー・3Dプロダクト可視化》
- 使用ソフト：《例：Illustrator / Premiere / Blender》

【厳守】
- 編集は MODELS 内の my_usage のみ。他のデータ・お気に入りロジック(FAV_KEY)・全レンダリング関数・リリース傾向ダッシュボードJS・CSS・AI追記プロトコルのコメント・外部依存なし構成は変更しない。
- 出力ファイル名は ai-model-library-private.html（公開版 index.html と取り違え防止）。公開先には上げない。

【進め方】
1. まず私のプロフィール理解を3〜5行で提示し「この理解で記入してよいか、編集点はあるか？」とだけ質問（モデル別の叩き台一覧は出さない）。
2. OKが出たら24件の my_usage を一括記入。私が不要としたモデルは空配列([])のまま。
3. 完成版を出力し「変更は my_usage のみ・他は無変更」を差分検証して報告。

【メモ】活用例に個人特定情報（住所・契約先固有名等）は含めない。
```

### B. 2回目以降：新リリースを追記する

新チャットに *対象HTML*（公開版 index.html または私用版）と *リリース告知テキスト* を添付して、以下を貼る：

```
添付HTMLにモデルの新リリースを追記してください。告知テキストも添付します。冒頭コメントの「AI追記プロトコル」に従うこと。

【編集範囲】
- MODELS への新規追加、CHANGELOG・META.session_verified の更新のみ。
- 既存モデルのデータ・お気に入りロジック(FAV_KEY)・全レンダリング関数・ダッシュボードJS・CSS・外部依存なし構成は変更しない。

【厳守ルール】
- official_url は各ベンダー公式の一次情報を検索で確認してから記入。確認できたら status:"official"、名称対応が断定できなければ "partial"、公式が無ければ "unverified"＋URL空。憶測でURLを埋めない。
- カテゴリは告知の接頭辞に合わせる。CATEGORIESに無い接頭辞は新キーを追加してよい。
- release_date は告知日。既存と重複するモデルは追加しない。
- ライセンス・料金・商用可否を断定する場合は一次情報で裏取りし、表記揺れがあれば明記。

【my_usage の扱い・取り違え防止】
- 公開版(index.html) → 追加モデルの my_usage は空配列([])のまま。
- 私用版(ai-model-library-private.html) → 《自分の制作領域》に沿って記入。実名（契約先・ブランド・案件）は出さない。
- どちらか不明なら編集前に確認すること。

【最後に】追加件数・各official_urlの確認ステータス・既存部分が無変更であることを差分検証して報告。
```
